Why Atheer

8 Reasons Atheer Is Different

Every mobile inference engine runs LLMs. Here’s why Atheer is the only one built for the NPU era — with predictive thermal management, hierarchical memory, structured output, model-level encryption, and autonomous agents, all in safe Rust.

⚡

#1 — NPU-First Multi-Backend

Atheer auto-probes NPU → GPU → CPU on both iOS and Android — CoreML/ANE, Metal, NNAPI, Vulkan — and selects the fastest available accelerator. Per-operation routing even sends decode to CPU in Eco mode to conserve GPU memory.

Competitors don’t: No other engine probes NPU before GPU before CPU on both platforms. llama.cpp has no NPU path. MLC LLM has none. ExecuTorch has Qualcomm-only. MLX Swift has none.

▦

#2 — Predictive Thermal Orchestration

Least-squares regression on live 1 Hz telemetry predicts temperature trajectory and pre-emptively downgrades inference mode before the OS thermal throttle hits — replacing the sudden 60% performance cliff with graceful degradation.

Competitors don’t: No mobile inference engine has thermal-aware operation. Every competitor runs flat-out until the OS forcibly throttles.

⊞

#3 — L1/L2/L3 KV Cache Hierarchy

A three-tier cache with promotion scoring and explicit handoff protocols. L1 holds the active session, L2 retains recent sessions for fast recall, and L3 stores compressed/archived context — designed for multi-turn agent sessions on memory-constrained devices.

Competitors don’t: Every other engine uses a single flat KV cache with no tiering, no promotion, and no cross-session handoff.

◆

#4 — Grammar-Constrained Decoding

A first-class GrammarConstraint trait implements a pushdown automaton that guarantees valid JSON output at every decode step — not a post-processing bolt-on, but a native part of the generation loop with thread-safe Send + Sync semantics.

Competitors don’t: Only llama.cpp has comparable GBNF grammar support, but it’s a C API bolt-on, not a first-class trait in a safe language.

⟳

#5 — Built-in Agent Loop

An autonomous Agent struct with configurable max_steps handles the generate → parse → tool-call → continue cycle natively. No need to build orchestration logic in the host app — Atheer ships the loop.

Competitors don’t: Every other mobile engine is a text-in-text-out API. You must build agent orchestration, context management, and tool-calling yourself.

◉

#6 — UniFFI Cross-Platform Bindings

One UniFFI definition generates idiomatic Swift packages for iOS and Kotlin packages for Android — from the same Rust codebase, with no platform-specific wrapper code or duplicate SDK maintenance.

Competitors don’t: llama.cpp exposes a raw C API requiring per-platform wrappers. ExecuTorch is C++-only. MLX Swift is Apple-exclusive.

◈

#7 — Safe Rust, Not C/C++

Rust’s ownership model guarantees no use-after-free, no data races, and no GC pauses. catch_unwind wraps ANE forward calls so accelerator panics don’t crash the host app. Memory safety at the FFI boundary, zero-cost abstractions everywhere.

Competitors don’t: Most mobile inference engines are written in C/C++ with manual memory management. Only MLX Swift shares Rust’s safety guarantees — but it’s Apple-only.

🔐

#8 — Model-Level Encryption

AES-256-GCM encryption for GGUF and .mlpackage model files at rest, with a three-strategy decryption pipeline — ServerDistributed keys, DeviceDerived via HKDF-SHA256, or Custom provider. Platform-native Keychain (AtheerKeychain.swift) and Android Keystore (KeyStoreManager.kt) wrappers for secure key storage. Offline encryption via the atheer-encrypt CLI tool.

Competitors don’t: No other open engine encrypts models at rest. llama.cpp, MLC, MLX Swift, and Google AI Edge all load models in plaintext. This is the only engine where you can prove model files never leave device storage unencrypted.

Features

Designed for on-device AI

Atheer combines hardware-accelerated inference with agentic capabilities — all in safe Rust, all running locally on mobile. Every feature maps to actual code in the workspace.

⚡

Multi-Backend GPU/NPU

Metal (iOS GPU via Candle), custom GLSL compute shaders compiled via naga for Android Vulkan, raw NDK FFI bindings for NNAPI (NPU/DSP), CoreML ANE compatibility detection — automatic probe with graceful CPU fallback via atheer-accel.

◆

Structured Output

Character-level grammar-constrained decoding via a 38-state Pushdown Automaton in atheer-orchestrator/src/grammar/ — guarantees valid JSON at every token, zero external dependencies, no post-processing.

⟳

Agentic Workflows

Draft-model speculative decoding with fallback NGram cache, tool-calling definitions with JsonGrammar integration, and autonomous agent loops — bring your own tools via the Agent trait in atheer-orchestrator.

▦

Adaptive Mode Switching

Turbo / Balanced / Eco modes switch in real-time using a predictive moving-average thermal model with trend detection — 1 Hz hardware telemetry from atheer-hardware feeds the Orchestrator for preemptive downgrades.

⊞

Memory Hierarchy

L1/L2/L3 KV cache with INT8/INT4 quantization and adaptive depth by context length. PagedAttention block manager (16-token blocks) eliminates fragmentation. Incremental checkpoint deltas enable sub-100ms save/restore.

◉

Performance-per-Watt

Criterion microbenchmarks (model-free) + perf-bench binary with model-dependent suites for throughput, latency, checkpoint, memory, energy, and thermal throttling. All in perf-bench/ and tests/benches/.

Platform Support

iOS & Android, every accelerator

Atheer auto-probes the best available backend in platform-specific priority order. NPU → GPU → CPU. Each backend is implemented as a Rust crate module — no black boxes.

iOS — Priority 1

CoreML / ANE

Apple Neural Engine via candle-coreml integration. ANECompatibility struct with per-layer heuristics, model size ceiling, and quantization whitelist. Falls back to Metal for incompatible architectures. catch_unwind protected.

ANE detection · macOS build
iOS — Priority 2

Metal GPU

GPU acceleration via candle_core::Device::Metal. Full tensor ops for prefill and decode in atheer-accel/src/metal.rs.

iOS 15+ · Candle Metal backend
Android — Priority 1

NNAPI NPU/DSP

Raw NDK FFI bindings with ~20 extern functions covering device discovery, graph construction (9 ops), compilation, and execution. Full pipeline in atheer-accel/src/nnapi_ndk.rs.

API 24+ · NDK r29+
Android — Priority 2

Vulkan GPU

Custom GLSL compute shaders (gemv.glsl, attention.glsl) compiled to SPIR-V via naga at build time. DP4A-style int8 matrix-vector multiply and flash attention in atheer-accel/src/vulkan.rs (~1120 lines).

API 26+ · custom compute shaders
Fallback

CPU

Always available fallback using Candle CPU backend. Graceful degradation when no accelerator is available — graceful, tested, documented.

Any platform · zero acceleration
Quick Start

From zero to inference

Build the workspace, run tests, and integrate into your iOS or Android app.

bash 
# Build the entire workspace
cargo build --workspace

# Run all ~390 tests
cargo test --workspace

# Download a test model (~350 MB)
scripts/download-test-model.sh

# Run integration tests with a real model
ATHEER_TEST_MODEL=./models/LFM2-700M-Q4_0.gguf \
  cargo test -p atheer-core -- --ignored

# Benchmark throughput
cargo run -p perf-bench -- \
  --model-path model.gguf \
  --batch-sizes 1,4,8
swiftiOS integration
import AtheerFFI

var config = AtheerConfig()
config.modelPath = "/models/llama.gguf"
config.adaptive = true

let engine = AtheerEngine(config: config)
try engine.initialize()

var request = GenerationRequest(prompt: "Hello")
let response = try engine.generateSync(request: request)
print(response.text)
kotlinAndroid integration
import uniffi.atheer_ffi.*

val config = AtheerConfig(
    modelPath = "/models/llama.gguf",
    modelId = null,
    draftModelPath = null,
    adaptive = true,
    maxTokens = 2048u,
    temperature = 0.7f,
    quantization = "q4_0",
    adaptiveHandoff = true,
    memoryBankSizeMb = 256u,
    standbyDraftPath = null
)

val engine = AtheerEngine(config)
engine.initialize()

val request = GenerationRequest(
    prompt = "Hello",
    maxTokens = 512u,
    temperature = 0.7f
)
val response = engine.generateSync(request)
println(response.text)
